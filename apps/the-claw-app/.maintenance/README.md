# Maintenance Reports Directory

This directory contains automated maintenance reports generated by the maintenance system.

## Reports Generated

### Dashboard Reports

- **`dashboard.html`** - Interactive HTML dashboard with visualizations
- **`dashboard.md`** - Markdown summary for quick reading
- **`dashboard.json`** - Machine-readable data for automation

### Individual Check Reports

- **`staleness.json`** - Files that haven't been updated recently
- **`versions.json`** - Dependency version check results
- **`documentation.json`** - Documentation health check results
- **`coverage.json`** - Test coverage analysis
- **`navigation.json`** - Documentation navigation structure

## Viewing Reports

### HTML Dashboard

Open in browser for interactive experience:

```bash
# macOS
open .maintenance/dashboard.html

# Linux
xdg-open .maintenance/dashboard.html

# Windows
start .maintenance/dashboard.html
```

### Markdown Summary

Quick view in terminal:

```bash
cat .maintenance/dashboard.md
```

### JSON Data

Parse programmatically:

```bash
# Get summary
jq '.summary' .maintenance/dashboard.json

# Get critical issues count
jq '.summary.criticalIssues' .maintenance/dashboard.json

# Get recommendations
jq '.recommendations[]' .maintenance/dashboard.json
```

## Automation

Reports are automatically generated:

- **Weekly:** Every Monday at 9 AM UTC (via GitHub Actions)
- **On-demand:** Manual workflow trigger
- **Local:** Run `npm run maintenance:dashboard`

## Report Retention

- **GitHub Actions:** 90 days (artifact retention)
- **Local:** Manual cleanup recommended
- **GitHub Pages:** Latest report always available
